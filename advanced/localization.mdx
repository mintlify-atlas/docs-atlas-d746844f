## How Localization Works

Nozmie's localization system consists of three main components:

### 1. Automatic Locale Detection

The addon automatically detects your game client's locale when it loads:

```lua
function Locale.GetLocale()
    return GetLocale() or "enUS"
end
```

This means you don't need to configure anything - Nozmie will use the appropriate language files based on your WoW client settings.

### 2. Localized String Lookup

All UI text, messages, and tooltips are stored in locale files located in `Locales/`:

- `Locales/enUS.lua` - English (fallback)
- `Locales/deDE.lua` - German
- `Locales/esES.lua` - Spanish
- `Locales/frFR.lua` - French
- `Locales/ruRU.lua` - Russian
- `Locales/koKR.lua` - Korean
- `Locales/ptBR.lua` - Portuguese
- `Locales/zhCN.lua` - Simplified Chinese
- `Locales/zhTW.lua` - Traditional Chinese

<Note>
If a translation is missing in your locale, Nozmie automatically falls back to English (enUS) to ensure the addon remains functional.
</Note>

### 3. Keyword Aliases

Nozmie supports localized keywords for detecting utility requests in chat. Each locale file can define custom keywords that match regional terminology:

```lua
keywordsByName = {
    ["Portal: Darnassus"] = {"darnassus", "darn"},
    ["Teleport: Stormwind"] = {"stormwind", "sw"},
    -- ... more keywords
}
```

This allows the addon to understand chat messages in your language, such as:
- "need port to org" (English)
- "brauche portal nach orgrimmar" (German)
- "necesito portal a ventormenta" (Spanish)

## Chat Detection Limitations

<Warning>
Nozmie currently uses English keywords for detecting dungeon names and some location names in chat. Full localized dungeon name detection is on the roadmap for future updates.
</Warning>

The addon will still work in non-English clients, but some location-based keywords may need to be in English for detection to work properly.

## Fallback System

Nozmie implements a robust fallback system to ensure the addon never breaks due to missing translations:

<Steps>
  <Step title="Check Current Locale">
    First, look for the string in your current locale (e.g., deDE for German)
  </Step>
  <Step title="Fall Back to English">
    If not found, retrieve the string from the English (enUS) locale
  </Step>
  <Step title="Use Key as Fallback">
    If still not found, display the key itself to prevent errors
  </Step>
</Steps>

```lua
function Locale.GetString(key, fallback)
    local locale = Locale.GetLocale()
    local data = GetLocaleTable(locale)
    local strings = data.strings or {}
    local value = strings[key]
    if value == nil then
        local en = Nozmie_Locales["enUS"]
        value = en and en.strings and en.strings[key] or nil
    end
    return value or fallback or key
end
```

This ensures Nozmie remains functional even if translations are incomplete.
