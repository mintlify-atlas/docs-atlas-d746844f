### Search Algorithm

From `UtilityUI.lua:94`:

```lua
local function MatchesSearch(item, query)
    if not query or query == "" then
        return true
    end
    
    -- Search name, spellName, destination, category
    local text = string.lower(table.concat({
        tostring(item.name or ""),
        tostring(item.spellName or ""),
        tostring(item.destination or ""),
        tostring(item.category or "")
    }, " "))
    
    if text:find(query, 1, true) then
        return true
    end
    
    -- Also search keywords
    if item.keywords then
        for _, keyword in ipairs(item.keywords) do
            if type(keyword) == "string" and 
               string.lower(keyword):find(query, 1, true) then
                return true
            end
        end
    end
    
    return false
end
```

<Steps>
  <Step title="Case-insensitive">
    All searches are converted to lowercase
  </Step>
  
  <Step title="Multiple fields">
    Searches name, spell name, destination, and category
  </Step>
  
  <Step title="Keyword matching">
    Also searches custom keywords for each entry
  </Step>
  
  <Step title="Cross-tab search">
    When searching, all tabs are searched simultaneously
  </Step>
</Steps>

<Info>
  When you type in the search box, it searches across **all categories**, not just the current tab.
</Info>

        width = 520  -- Fallback
    end

    end

## Data Caching

### Building the Cache

From `UtilityUI.lua:278`:

```lua
local function BuildDataCache()
    dataCache = {}
    for _, item in ipairs(_G.Nozmie_Data) do
        if (IsUtilityEntry(item) or 
            IsTeleportEntry(item) or 
            IsHearthstoneEntry(item)) and 
           IsUsableUtility(item) then
            table.insert(dataCache, item)
        end
    end
    
    -- Sort alphabetically by name
    table.sort(dataCache, function(a, b)
        return ConfigHelpers.GetEntryName(a) < 
               ConfigHelpers.GetEntryName(b)
    end)
end
```

<Steps>
  <Step title="Filter by category">
    Only includes utilities, teleports, and hearthstones
  </Step>
  
  <Step title="Check usability">
    Verifies you have learned the spell or own the item
  </Step>
  
  <Step title="Sort alphabetically">
    Entries are sorted by display name
  </Step>
  
  <Step title="Cache for performance">
    Built once on frame show, reused for filtering
  </Step>
</Steps>

### Usability Check

From `UtilityUI.lua:260`:

```lua
local function IsUsableUtility(item)
    if item.itemID then
        if PlayerHasToy and PlayerHasToy(item.itemID) then
            return true
        end
        if C_Item and C_Item.GetItemCount then
            return C_Item.GetItemCount(item.itemID, true, false, false) > 0
        end
        return false
    end
    
    if item.spellID and (IsSpellKnown(item.spellID) or 
                         IsPlayerSpell(item.spellID)) then
        return true
    end
    
    return false
end
```

## Performance Optimization

### Lazy Button Creation

Buttons are created on-demand and reused:

```lua
local function EnsureButton(index)
    if buttons[index] then
        return buttons[index]  -- Reuse existing
    end
    -- Create new button only if needed
    local button = CreateFrame(...)
    buttons[index] = button
    return button
end
```

### Efficient Filtering

From `UtilityUI.lua:304`:

```lua
local function BuildFilteredData()
    filteredData = {}
    
    -- If searching, search all items
    if query and query ~= "" then
        for _, item in ipairs(dataCache) do
            if MatchesSearch(item, query) then
                table.insert(filteredData, item)
            end
        end
    else
        -- Otherwise filter by active tab
        for _, item in ipairs(dataCache) do
            if ItemMatchesTab(item, selectedTab) then
                table.insert(filteredData, item)
            end
        end
    end
end
```

<Info>
  The utility page uses efficient data structures and lazy loading to maintain smooth performance even with hundreds of teleport options.
</Info>
