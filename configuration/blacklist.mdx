## Overview

The blacklist feature allows you to filter out messages containing specific words or phrases. When a blacklisted word is detected in a message, Nozmie will not process that message for teleport or utility detection.

<Warning>
  Blacklist filtering happens **before** any other detection logic. If a message contains a blacklisted word, it is immediately rejected and no matches will be returned.
</Warning>

## How It Works

The blacklist operates on a simple substring matching system:

1. Messages are converted to lowercase
2. Each blacklisted word is checked against the lowercase message
3. If any blacklisted word is found, the entire message is rejected
4. No banner is shown and no utilities are detected

<Info>
  Blacklist matching is case-insensitive and uses simple substring matching (`string.find` with plain text mode).
</Info>

## Configuration

### Via Settings Panel

1. Open settings: **ESC** → **Options** → **Addons** → **Nozmie**
2. Navigate to the **Blacklist** section
3. Click the **Edit Blacklisted Words** button
4. Enter words separated by commas in the dialog
5. Click **Save**

<ParamField path="NozmieDB.blacklistedWords" type="string" default='""'>
  Comma-separated list of blacklisted words
  
  **Example:**
  ```lua
  NozmieDB.blacklistedWords = "spam, advertisement, selling, wtb, wts"
  ```
</ParamField>

### Programmatically

```lua
local Settings = _G.Nozmie_Settings

-- Get current blacklist
local blacklist = Settings.Get("blacklistedWords")
-- Returns: "spam, advertisement, selling"

-- Set new blacklist
Settings.Set("blacklistedWords", "word1, word2, word3")

-- Clear blacklist
Settings.Set("blacklistedWords", "")

-- Add a word to existing blacklist
local current = Settings.Get("blacklistedWords") or ""
if current ~= "" then
    current = current .. ", newword"
else
    current = "newword"
end
Settings.Set("blacklistedWords", current)
```

## Blacklist Logic

The blacklist check is performed in `Detector.lua:124-134`:

```lua
if Settings then
    local blacklist = Settings.Get("blacklistedWords") or ""
    if blacklist ~= "" then
        -- Parse comma-separated words
        for word in blacklist:gmatch("([^,]+)") do
            -- Trim whitespace and convert to lowercase
            local trimmedWord = word:match("^%s*(.-)%s*$"):lower()
            if trimmedWord ~= "" and lowerMessage:find(trimmedWord, 1, true) then
                -- Word found - reject entire message
                return {}
            end
        end
    end
end
```

### Processing Steps

1. **Get blacklist string:** `Settings.Get("blacklistedWords")`
2. **Parse words:** Split by comma using `gmatch("([^,]+)")`
3. **Trim whitespace:** Remove leading/trailing spaces with `match("^%s*(.-)%s*$")`
4. **Convert to lowercase:** `.lower()`
5. **Search message:** Use `find(trimmedWord, 1, true)` for plain text matching
6. **Return early:** If found, return empty table (no matches)

<Note>
  The `true` parameter in `find()` enables plain text mode, disabling Lua pattern matching. This means special characters like `.` and `*` are treated literally.
</Note>

## Popup Dialog

The blacklist editor uses a custom popup dialog defined in `Settings.lua:400-429`:

```lua
StaticPopupDialogs["NOZMIE_BLACKLIST_INPUT"] = {
    text = "Set blacklisted words (comma-separated):",
    button1 = "Save",
    button2 = "Cancel",
    hasEditBox = true,
    editBoxWidth = 260,
    OnShow = function(self)
        -- Pre-fill with current blacklist
        local current = Settings.Get("blacklistedWords") or ""
        editBox:SetText(current)
        editBox:SetAutoFocus(true)
        editBox:HighlightText()
    end,
    OnAccept = function(self)
        -- Save new blacklist
        local value = editBox:GetText() or ""
        Settings.Set("blacklistedWords", value)
        print("|cff00ff00Nozmie:|r Blacklist updated to: |cffFFFFFF" .. value .. "|r")
    end,
    timeout = 0,
    whileDead = true,
    hideOnEscape = true
}
```

### Dialog Features

- Pre-fills with current blacklist when opened
- Auto-focuses and highlights text for easy editing
- 260 pixel wide edit box for comfortable editing
- Works while dead
- Dismissible with Escape key
- Shows confirmation message on save

### Testing Your Blacklist

```lua
-- Test blacklist filtering
local function testBlacklist(message)
    local Settings = _G.Nozmie_Settings
    local Detector = _G.Nozmie_Detector
    
    local matches = Detector.FindMatchingTeleports(message, "TestSender")
    
    if #matches == 0 then
        print("Message rejected (blacklisted or no matches)")
    else
        print(string.format("Found %d matches", #matches))
    end
end

-- Test with blacklisted word
testBlacklist("Can someone portal to Orgrimmar? WTS gold")
-- Output: Message rejected (blacklisted or no matches)

-- Test without blacklisted word
testBlacklist("Can someone portal to Orgrimmar?")
-- Output: Found 1 matches
```

## Case Sensitivity

Blacklist matching is **case-insensitive**:

```
Blacklist: "spam, SELLING, WtB"

Messages filtered:
- "spam in chat"
- "SPAM in chat"
- "Spam in chat"
- "selling items"
- "SELLING items"
- "Selling items"
- "wtb mats"
- "WTB mats"
- "WtB mats"
```

## Whitespace Handling

Leading and trailing whitespace is automatically trimmed:

```
Blacklist: "  spam  ,   selling  ,  wtb  "

Normalized to: "spam", "selling", "wtb"
```

<Info>
  You can use spaces after commas for readability. They're automatically removed during parsing.
</Info>

## Substring Matching

Blacklist uses **substring matching**, not whole-word matching:

```
Blacklist: "port"

Messages filtered:
- "port to Org?" ✓
- "portal please" ✓
- "teleport me" ✓
- "support needed" ✓ (unintended match)
- "report this player" ✓ (unintended match)
```

<Warning>
  Be careful with short words. "port" will also match "portal", "teleport", "support", "report", etc.
</Warning>

## Performance Considerations

The blacklist check is very efficient:

- **Early rejection:** Happens before expensive detection logic
- **Simple string matching:** No regex or complex parsing
- **Minimal overhead:** Only processes when blacklist is non-empty

```lua
-- Benchmark: Processing 1000 messages
-- With blacklist: ~0.5ms
-- Without blacklist: ~150ms (full detection)
-- Speedup when filtering: ~300x
```

<Info>
  Using the blacklist actually **improves** performance by rejecting unwanted messages early in the processing pipeline.
</Info>

## Blacklist vs. Suppression Filters

**Blacklist** and **Suppression Filters** serve different purposes:

| Feature | Blacklist | Suppression Filters |
|---------|-----------|---------------------|
| Filters by | Message content | Utility category |
| Scope | All messages | Detected utilities |
| Matching | Substring | Category tags |
| When applied | Before detection | After detection |
| Performance | High (early rejection) | Moderate |
| Use case | Filter spam/keywords | Control detection types |

<Note>
  Use **blacklist** to filter messages containing specific words. Use **suppression filters** to disable detection of specific utility categories.
</Note>

## Troubleshooting

### Blacklist Not Working

1. Verify the blacklist is saved:
   ```lua
   /dump NozmieDB.blacklistedWords
   ```

2. Check for typos in blacklisted words

3. Ensure words are comma-separated

4. Test with `/noz blacklist` to view current list

### False Positives

1. Review your blacklisted words for overly-broad terms
2. Remove short words that match unintended content
3. Use more specific phrases

### Can't Edit Blacklist

1. Ensure you're in the correct settings panel
2. Try using slash command instead: `/noz blacklist word1, word2`
3. Check for UI errors: `/console scriptErrors 1`
